Implementation note: To make `i18n/request.ts` testable, extract the locale resolution into a named export:
```typescript
export function resolveLocale(cookieValue?: string): string {
  return cookieValue || 'vi';
}
```
Then use it in `getRequestConfig`:
```typescript
const locale = resolveLocale(store.get('NEXT_LOCALE')?.value);
```
This keeps the cookie reading in the Next.js server context while making the defaulting logic unit-testable.
