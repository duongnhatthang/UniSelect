Modify `.github/workflows/scrape-peak.yml` to add caching for both PaddleOCR models and Playwright browsers. The exact changes:

1. Add `PADDLE_PDX_MODEL_SOURCE: BOS` as a job-level `env:` on the `scrape` job (NOT step-level). This prevents PaddleOCR from remote model verification after cache restore.

2. After `npm ci` and before the current "Install Playwright browsers" step, add a Playwright browser cache step:
```yaml
- name: Cache Playwright browsers
  id: playwright-cache
  uses: actions/cache@v4
  with:
    path: ~/.cache/ms-playwright
    key: playwright-${{ runner.os }}-${{ hashFiles('package-lock.json') }}
    restore-keys: |
      playwright-${{ runner.os }}-
```

3. Replace the unconditional "Install Playwright browsers" step with TWO conditional steps:
```yaml
- name: Install Playwright browsers
  if: steps.playwright-cache.outputs.cache-hit != 'true'
  run: npx playwright install chromium --with-deps

- name: Install Playwright OS dependencies
  if: steps.playwright-cache.outputs.cache-hit == 'true'
  run: npx playwright install-deps chromium
```
CRITICAL: On cache hit, OS shared libraries (libglib, libnss) still need reinstalling via `install-deps`. Skipping this causes Chromium launch failures.

4. After the `actions/setup-python` step and before "Install PaddleOCR", add a PaddleOCR model cache step (ported from ci-ocr.yml):
```yaml
- name: Cache PaddleOCR models (~/.paddlex)
  uses: actions/cache@v4
  with:
    path: ~/.paddlex
    key: paddleocr-models-${{ runner.os }}-${{ hashFiles('scripts/ocr_table.py') }}
    restore-keys: |
      paddleocr-models-${{ runner.os }}-
```

5. Keep the existing "Install PaddleOCR" (`pip install paddleocr`) and "Warm up PaddleOCR models" steps unchanged — they are fast with pip cache and the warm-up is a no-op when models are cached.

6. Keep all other steps (checkout, setup-node, npm ci, run scraper) unchanged. Preserve the `if:` condition on the job, the matrix strategy, and the env vars on the run step.
