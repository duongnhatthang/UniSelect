Create a new file `.github/workflows/supabase-keepalive.yml` with these exact contents:

```yaml
name: Supabase keep-alive

on:
  schedule:
    - cron: '0 10 */5 * *'   # Every 5 days at 10:00 UTC — within 7-day pause window
  workflow_dispatch:

jobs:
  ping:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - name: Ping Supabase database
        run: npx tsx -e "
          import postgres from 'postgres';
          const sql = postgres(process.env.DATABASE_URL!, { prepare: false });
          const result = await sql\`SELECT 1 as ok\`;
          console.log('keep-alive: database is active', result);
          await sql.end();
        "
        env:
          DATABASE_URL: ${{ secrets.DATABASE_URL }}
```

Key design decisions:
- Uses `npx tsx -e` with inline TypeScript to avoid creating a separate script file. This is consistent with the project's `npx tsx` pattern used in all other workflows.
- Uses `postgres` package (postgres.js) which is the project's actual DB driver — NOT `pg` which is not installed.
- Sets `prepare: false` matching the existing `lib/db/index.ts` pattern (required for Supabase Supavisor transaction pool mode).
- Cron `*/5` in day-of-month = days 1, 6, 11, 16, 21, 26, 31. Maximum gap is 6 days (day 26 to day 1 in short months like February) — safely within the 7-day Supabase pause window.
- `workflow_dispatch` allows manual trigger if a scheduled run fails.
- Runs `npm ci` to ensure `postgres` package is available (it is a direct dependency).
