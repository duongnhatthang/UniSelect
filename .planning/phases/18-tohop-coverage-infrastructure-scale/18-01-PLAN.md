    2. Inside `createCheerioAdapter`, in the `$('table').each()` callback, AFTER headers are determined and column indices found, add a wide-table guard BEFORE the existing narrow-table `allRows.slice(1).each()`:

    ```typescript
    if (config.wideTable) {
      // Detect to hop columns: headers matching /^[A-D]\d{2}$/i
      const tohopCols: Array<{ idx: number; code: string }> = [];
      headers.forEach((h, idx) => {
        const cleaned = h.trim().toUpperCase();
        if (/^[A-D]\d{2}$/.test(cleaned)) {
          tohopCols.push({ idx, code: cleaned });
        }
      });

      if (tohopCols.length === 0) return; // not a wide-table, skip this table

      // Use scoreKeywords to verify this is a cutoff table (safety guard)
      // For wide-tables, scoreKeywords presence in ANY header confirms context
      const hasScoreContext = headers.some((h) =>
        config.scoreKeywords.some((kw) => h.includes(kw))
      );
      // Also accept tables where to hop columns exist (some wide-tables omit "diem chuan" header)

      allRows.slice(1).each((_, tr) => {
        const cells = $(tr).find('td').map((_, td) => $(td).text().trim()).get();
        if (cells.length < 3) return;
        const majorCode = codeIdx !== -1 ? cells[codeIdx] : '';
        if (!majorCode || !/^\d/.test(majorCode)) return;

        for (const col of tohopCols) {
          const scoreRaw = cells[col.idx] ?? '';
          if (!scoreRaw || !/\d/.test(scoreRaw)) continue; // empty = not offered
          rows.push({
            university_id: config.id,
            major_raw: majorCode,
            tohop_raw: col.code,
            year,
            score_raw: scoreRaw,
            source_url: url,
          });
        }
      });
      return; // wide-table path handled, don't fall through to narrow path
    }
    ```

    3. The existing narrow-table path (starting at `if (scoreIdx === -1) return;`) remains UNCHANGED after the wide-table guard.

    4. IMPORTANT: The wide-table guard skips the `scoreIdx === -1` check because wide-tables have no single score column. The `codeIdx` (major column) detection still works via `majorKeywords`.

    5. IMPORTANT: The `return` at the end of the wide-table block exits the current `.each()` iteration (for THIS table), not the outer `scrape()` function. The 0-rows guard at the end of `scrape()` still fires if no rows were produced.
