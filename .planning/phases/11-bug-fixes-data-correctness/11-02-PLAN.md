    **FIX-05: Timer cleanup in lib/db/timeout.ts**

    Replace the entire `withTimeout` function with:
    ```typescript
    export function withTimeout<T>(promise: Promise<T>, ms: number): Promise<T> {
      let timerId: ReturnType<typeof setTimeout>;
      const timeout = new Promise<never>((_, reject) => {
        timerId = setTimeout(() => reject(new Error('DB_TIMEOUT')), ms);
      });
      return Promise.race([promise, timeout]).finally(() => clearTimeout(timerId!));
    }
    ```

    Key change: capture `timerId`, chain `.finally(() => clearTimeout(timerId!))` on `Promise.race()`. The `.finally()` fires on both resolution and rejection, ensuring the timer is always cleared.

    **Add timer cleanup test in tests/api/helpers-timeout.test.ts**

    Add to the existing `withTimeout` describe block:

    it('clears the timeout timer after promise resolves (no leak)', async () => {
