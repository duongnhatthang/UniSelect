    - Test 2: A page with title containing "diem chuan" scores at least TITLE_WEIGHT (2)
    - Test 3: A page with h2 heading "Diem chuan 2024" scores at least HEADING_WEIGHT (1)
    - Test 4: A page with a table whose headers include "Ma nganh" and "Diem chuan" scores TABLE_WEIGHT (5) from table detection
    - Test 5: A full cutoff page (URL slug + title + heading + table) scores at least 11 (3+2+1+5)
    - Test 6: A generic news page mentioning "diem chuan" once in body text but no table/heading match scores 0 (body text is NOT scored, only title/URL/heading/table)
    - Test 7: reasons array contains specific matched signal strings (e.g., "url:diem-chuan", "title:diem chuan", "table:score-columns-detected")
    - Test 8: Empty HTML page scores 0 with empty reasons array
  </behavior>
  <action>
1. Create `tests/scraper/discovery/keyword-scorer.test.ts` with the 8 test cases above. Use cheerio's `load()` to create the CheerioAPI from HTML strings inline in each test. Import `scorePageForCutoffs` from `../../../lib/scraper/discovery/keyword-scorer`. Each test constructs a minimal HTML string and asserts the returned `{ score, reasons }`.

2. Run tests — they MUST fail (RED phase).

3. Create `lib/scraper/discovery/keyword-scorer.ts`:
   - Import `CheerioAPI` type from `cheerio`
   - Import keyword arrays and weights from `./constants`
   - Export function `scorePageForCutoffs(url: string, $: CheerioAPI): { score: number; reasons: string[] }`
   - Implementation:
     a. Check URL against URL_SLUG_KEYWORDS — add URL_SLUG_WEIGHT per match, push `url:${keyword}` to reasons
     b. Check `$('title').text().toLowerCase()` against PAGE_TITLE_KEYWORDS — add TITLE_WEIGHT per match, push `title:${keyword}` to reasons
     c. Check `$('h1, h2, h3').text().toLowerCase()` for each heading against HEADING_KEYWORDS — add HEADING_WEIGHT per match (break after first match per heading), push `heading:${keyword}` to reasons
     d. Check tables: for each `$('table')`, get text of first 10 th/td cells, check against TABLE_HEADER_KEYWORDS. If any table matches, add TABLE_WEIGHT once, push `table:score-columns-detected` to reasons
   - Return `{ score, reasons }`

4. Run tests — they MUST pass (GREEN phase).

5. Run full test suite to verify no regressions: `npx vitest run`
