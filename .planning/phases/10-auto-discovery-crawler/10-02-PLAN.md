Build the CheerioCrawler-based discovery script and integration test it with MSW fake university homepages.   <action>
Create `scripts/discover.ts` as a standalone script that can be run with `npx tsx scripts/discover.ts`.

The script must:

1. **Export a `runDiscover` function** for testing (not just a main script):
   ```typescript
   export async function runDiscover(
     startUrls: { url: string; universityId: string }[],
     options?: { useMemoryStorage?: boolean }
   ): Promise<DiscoveryCandidate[]>
   ```

2. **CheerioCrawler configuration** (per research Pattern 1):
   - `sameDomainDelaySecs: 2` — per-domain rate limit (SCRP-05)
   - `respectRobotsTxtFile: { userAgent: 'UniSelectBot/1.0' }` — robots.txt compliance (SCRP-05)
   - `maxConcurrency: 1` — polite single-thread crawling
   - `maxRequestsPerCrawl: 50` — safety cap per run
   - When `options.useMemoryStorage` is true, create `new MemoryStorage({ persistStorage: false })` and pass via `new Configuration({ storageClient: storage })` as second arg to CheerioCrawler constructor. This is REQUIRED for tests to avoid filesystem writes.
   - When not in test mode, set env `CRAWLEE_STORAGE_DIR` to a temp directory (use `os.tmpdir()` + `/crawlee-discover`) or use MemoryStorage anyway.

3. **requestHandler** implementation:
   - Call `scorePageForCutoffs(request.url, $)` from keyword-scorer.ts
   - If score > 0, push `{ url: request.url, universityId: request.userData.universityId, score, reasons }` to candidates array
   - Call `enqueueLinks` with:
     - `strategy: 'same-hostname'`
     - `globs: ['**/diem-chuan*', '**/diem-trung-tuyen*', '**/tuyen-sinh*', '**/xet-tuyen*', '**/thong-bao*', '**/tin-tuyen-sinh*', '**/ket-qua-trung-tuyen*']`
     - `exclude: ['**/*.pdf', '**/*.jpg', '**/*.png', '**/*.gif', '**/*.doc', '**/*.docx', '**/*.xls', '**/*.xlsx']`
     - `userData: request.userData` to propagate universityId

4. **Start URL construction:**
   - Read `scrapers.json` from project root
   - For each entry, extract the homepage URL (parse the `url` field to get the origin/homepage, not the deep cutoff page itself)
   - Create start URLs as `[{ url: homepage, userData: { universityId: entry.id } }]`
   - The `runDiscover` function accepts pre-built startUrls for testing (bypasses scrapers.json reading)

5. **Output:**
   - Filter candidates where `score >= SCORE_THRESHOLD`
   - Sort descending by score
   - Deduplicate by URL (keep highest score if same URL found via different paths)
   - Write to `discovery-candidates.json` with `JSON.stringify(output, null, 2)`
   - Log summary: `Discovered {N} candidates from {M} universities`

6. **Main block:**
   - Only runs when script is executed directly (not imported by tests)
   - Reads scrapers.json, builds start URLs from unique homepage origins, calls runDiscover, writes output file
   - Use `import.meta.url` / `process.argv[1]` check or wrap in an IIFE guarded by `if (process.argv[1]?.endsWith('discover.ts'))` pattern

CRITICAL: The script must NEVER write to scrapers.json. All output goes to discovery-candidates.json only.
