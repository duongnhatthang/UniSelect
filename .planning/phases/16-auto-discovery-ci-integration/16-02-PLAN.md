**Then create scripts/apply-discovery.ts:**

The script must:
1. Export a pure `applyDiscovery(entries: ScraperEntry[], candidates: DiscoveryCandidate[]): { entries: ScraperEntry[]; patched: number }` function for testing
2. Build a `bestByUni` Map from candidates (first occurrence wins since candidates are pre-sorted by score descending)
3. For each entry in scrapers.json:
   - GUARD: if `entry.scrape_url !== null` then skip (never overwrite verified URLs)
   - GUARD: if `entry.adapter_type === 'skip'` then skip (never touch excluded entries)
   - If a candidate exists for `entry.id`, set `entry.scrape_url = candidate.url` AND set `entry.adapter_type = 'cheerio'` (CRITICAL: registry.ts line 38-40 uses factory_config with createCheerioAdapter for entries that have factory_config, and "pending" entries already have factory_config, but "pending" is ambiguous -- setting to "cheerio" makes intent explicit)
   - Log each patch to console: `[apply] ${entry.id}: set scrape_url = ${candidate.url} (score ${candidate.score})`
4. Main block (guarded by `process.argv[1]` check like discover.ts):
   - Read candidates from `process.argv[2]` or default to `discovery-candidates.json` in cwd
   - Read scrapers.json from cwd
   - Call `applyDiscovery(entries, candidates)`
   - Write updated entries back to scrapers.json with `JSON.stringify(entries, null, 2)` + trailing newline
   - Print summary: `[apply] Patched ${patched} entries. scrapers.json updated.`
